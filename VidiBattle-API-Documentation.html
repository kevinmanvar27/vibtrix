Vibtrix API Documentation

This document provides comprehensive documentation for all APIs available in the Vibtrix platform.

Authentication

Vibtrix uses JWT (JSON Web Token) for authentication with two types of tokens:

  • Access Token: Short-lived token (1 hour) for API access
  • Refresh Token: Long-lived token (7 days) to obtain new access tokens
POST /api/auth/token

Generate Authentication Tokens

Authenticates a user and returns access and refresh tokens.

Headers

Name Required Description
Content-Type Yes application/json

Request Body

{
  "username": "username",
  "password": "password"
}

Success Response (200 OK)

{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "user_id",
    "username": "username",
    "displayName": "Display Name",
    "avatarUrl": "https://example.com/avatar.jpg",
    "role": "USER"
  }
}

Error Responses

  • 400 Bad Request: Missing username or password
  • 401 Unauthorized: Invalid credentials
  • 403 Forbidden: Account does not have permission to log in
POST /api/auth/refresh

Refresh Access Token

Generates a new access token using a valid refresh token.

Headers

Name Required Description
Content-Type Yes application/json

Request Body

{
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Success Response (200 OK)

{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Error Responses

  • 400 Bad Request: Missing refresh token
  • 401 Unauthorized: Invalid or expired token
POST /api/auth/revoke

Revoke Refresh Token (Logout)

Invalidates a refresh token, effectively logging the user out.

Headers

Name Required Description
Content-Type Yes application/json

Request Body

{
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}

Success Response (200 OK)

{
  "message": "Token revoked successfully"
}

Error Responses

  • 400 Bad Request: Missing refresh token
  • 401 Unauthorized: Invalid token

Using Authentication Tokens

For all authenticated API requests, include the access token in the Authorization header:

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
